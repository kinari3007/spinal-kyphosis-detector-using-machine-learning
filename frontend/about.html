About This Project

Understanding the technology and methodology behind our AI prediction tool

Project Overview

This application demonstrates the application of machine learning in healthcare by predicting the likelihood of kyphosis development following spinal surgery. Built as an educational tool, it showcases how artificial intelligence can assist in medical risk assessment while emphasizing the importance of professional medical judgment.

The project serves as a portfolio demonstration of full-stack development skills, combining modern web technologies with machine learning to create an intuitive, responsive interface for exploring medical AI applications.

Model Performance Metrics

Our Decision Tree model was evaluated on a held-out test set representing 20% of the available data. Below are the key performance metrics:

Metric Value Description

Understanding the Metrics

  • Accuracy: Percentage of correct predictions overall
  • Precision: Of all "present" predictions, how many were actually correct
  • Recall: Of all actual "present" cases, how many were correctly identified
  • F1-Score: Harmonic mean of precision and recall, balancing both metrics

Model Characteristics

  • Algorithm: Decision Tree Classifier
  • Features: Age, vertebrae count, surgery start level
  • Training Size: ~2,400 patient records
  • Validation: Stratified train-test split (80/20)

Dataset & Training

Data Characteristics

  • Size: 3,000+ patient records
  • Features: 3 numerical inputs (age, vertebrae count, start level)
  • Target: Binary classification (present/absent)
  • Balance: ~77% absent, ~23% present cases
  • Source: Synthetic data based on real clinical patterns

Training Process

  • Preprocessing: Data validation and encoding
  • Split: 80% training, 20% testing with stratification
  • Algorithm: Decision Tree with hyperparameter tuning
  • Validation: Cross-validation for robust performance estimation
  • Metrics: Comprehensive evaluation across multiple measures

Limitations & Important Considerations

Critical Limitations

  • Educational Purpose Only: This tool is designed for learning and demonstration, not clinical use
  • Synthetic Data: While based on real patterns, the training data is artificially generated
  • Limited Scope: Model only considers 3 variables; real medical decisions involve many more factors
  • No Real-Time Updates: Model is static and doesn't learn from new cases
  • Population Bias: May not generalize to all patient populations or surgical techniques

Professional Medical Guidance Required

Real medical decisions must always incorporate clinical expertise, patient history, imaging studies, physical examination findings, and current best practices. This tool cannot and should not replace the comprehensive assessment provided by qualified healthcare professionals.

Ethical Considerations

AI in healthcare must be developed and deployed responsibly, with transparency about limitations, potential biases, and appropriate use cases. This project serves as an example of how to build educational tools while maintaining clear boundaries about their intended applications.
